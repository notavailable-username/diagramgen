    As you plan, use self-reflection to iteratively refine any areas of improvement in the plan.

    *Steps*: [PLEASE OUTPUT THE DRAFTS AND EVALUATIONS]
        1. Analyze the question and identify all values and their respective units in the question.

        3. WRITE out a draft of the arguments to be passed to the create_bar_model tool. There is no need to write it in code format as long as it's clear.
        4. Evaluate the draft using the following steps and criterion.
            1. All relationships stated in the question can be seen on the model, implicitly shown does not count as appearing on the model.
            2. All values in the question must appear on the model, implicitly shown does not count as appearing on the model. (There are two methods of labelling a bar: 1. within the segment or 2. through a horizontal brace)
            3. Values that did not come from the question (including values derived through or after solving the question) MUST NOT be labelled on the model.
            4. The number of bars must be kept to a minimum (normally equivalent to the number of entities).
            5. Difference (between entities/before-after) relationship should be shown by cutting out the segment and it should be towards the right of the model accordingly (PLEASE TAKE NOTE: vice versa segments of EQUAL length SHOULD usually be left aligned for alignment across bars, and do ensure equal length segments are ALIGNED as much as possible).
            6. Proportional relationships should be shown by cutting longer bar into n segments if the proportional relationship is n times. (e.g. A is n times of B means A would be divided into n segments of length B)
            7. If there are both difference and proportional relationships to be shown, it is difficult to divide the bar. Hence, priority should be given to showing the DIFFERENCE relationship via segments and the proportional relationship need not be shown via segments and can be labelled via a horizontal brace using 1u, 2u 3u etc (representing n units).

(e.g. Bob has 20 more apples than Mary and ate 50 apples. -> segment of 20 and 50 overlap, hence the 50 needs to be divided into 30 and 50-30=20, after which a horizontal brace can be used to label the 50 (as it spans two segments), while the 20 can be labelled using both methods)

            11. If there are total/sum relationships across bars, please show them through a vertical brace that spans across the relevant bars. DO NOT FORGET to label the total values.
            12. DO NOT use vertical braces to show differences (differences include any kind of differences be it derived or direct) as it is visually confusing and not intuitive.
            13. Do not label the same information repeatedly.
            14. DO NOT oversimplify the model and ensure it shows the required information. If there are missing information, THAT IS A RED FLAG and it should not be ignored. More thinking should be done on how to integrate the information. The model should be a reflection of what the question is trying to bring across.
